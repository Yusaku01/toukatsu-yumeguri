# 東葛湯めぐりマップ

東葛地域とその周辺で、日帰り温浴施設を探し、訪問の記録を育て、共有できる湯めぐりガイド。口コミ、ログイン、投稿、独自DBを持つコミュニティサービスではなく、静的に管理された施設情報と端末内の個人記録を組み合わせる。

## Language

**温浴施設**:
一般利用できる日帰り入浴先。スーパー銭湯、日帰り温泉、浴槽利用を主目的にできるサウナ併設施設を含む。
_Avoid_: 店舗, スポット, サウナ施設

**掲載対象**:
東葛湯めぐりマップに掲載してよい温浴施設。営業時間、料金、休館日などの変動情報を網羅する対象ではない。
_Avoid_: 登録施設, 加盟施設

**東葛地域**:
松戸市、柏市、流山市、我孫子市、野田市、鎌ケ谷市を中心とする主対象エリア。
_Avoid_: 千葉全域, 首都圏全域

**周辺候補**:
東葛地域の外側にあるが、東葛地域から足を伸ばす候補として扱う温浴施設。主対象ではないため、東葛地域と同じ密度で網羅するとは限らない。
_Avoid_: 対象外施設, 番外編

**湯めぐり記録**:
利用者が自分の端末で保持する訪問・関心・進捗の個人記録。公開投稿、口コミ、アカウントに紐づく履歴ではない。
_Avoid_: ユーザー投稿, 口コミ, アカウント履歴

**行った施設**:
利用者が訪問済みとして湯めぐり記録に入れた温浴施設。訪問の証明やレビューを意味しない。
_Avoid_: チェックイン, 訪問レビュー

**気になる施設**:
利用者が後で見たい候補として湯めぐり記録に入れた温浴施設。公開ブックマークではない。
_Avoid_: お気に入り, フォロー

**湯めぐりガイド**:
温浴施設を探すための地図、一覧、施設ページ、用途別まとめ、湯めぐり記録をまとめた体験全体。
_Avoid_: 口コミサイト, SNS, 予約サイト

**施設共有URL**:
既存の地図や一覧の中で、特定の温浴施設を直接開くための深いリンク。施設単体を説明する独立ページではなく、SNSやDMで「この施設を見て」と共有するための入口。共有の正規入口は地図で、一覧は比較のための補助入口とする。
_Avoid_: 施設詳細ページ, 口コミページ

## Example Dialogue

Dev: 「施設ページを作るなら、口コミも載せますか？」

Domain expert: 「載せません。施設ページは温浴施設を選ぶための湯めぐりガイドで、口コミサイトではありません。」

Dev: 「では、利用者が訪問済みにした情報は公開されますか？」

Domain expert: 「公開されません。それは湯めぐり記録で、この端末に残る個人記録です。共有する場合も、本人が進捗やスタンプを切り出して共有する形です。」

Dev: 「東葛地域外の施設はどう扱いますか？」

Domain expert: 「東葛地域から行きやすいものは周辺候補として扱います。ただし、東葛地域そのものと同じ網羅性は約束しません。」

Dev: 「特定の施設を共有したい場合は、施設ページを作る必要がありますか？」

Domain expert: 「まずは施設共有URLで十分です。共有されたURLを開くと地図や一覧の中でその温浴施設が開く、という扱いで、料金や営業時間を抱える独立ページとは分けます。」

Dev: 「施設共有URLは地図と一覧のどちらを正規にしますか？」

Domain expert: 「地図を正規入口にします。共有された人がまず知りたいのは位置と施設の概要で、一覧は比較したいときの補助入口です。」
