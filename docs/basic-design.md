# 東葛湯めぐりマップ 基本設計

## 目的

東葛地域とその周辺で、今日行きたい日帰り温浴施設をすばやく選べる地図サービスにする。

口コミ、ログイン、投稿、独自DBは持たず、施設情報はリポジトリ内の静的データとして管理する。運用コストを小さくし、GitHubから施設情報の追加・修正に参加できる形を優先する。

## 対象

主な利用者は、東葛地域に住んでいる人、首都圏から近場の温浴施設を探す人、施設情報をGitHubで修正したい人。

対象エリアは松戸市、柏市、流山市、我孫子市、野田市、鎌ケ谷市を中心にする。東葛地域周辺の吉川市、船橋市、千葉市などの周辺施設も、データに含まれる場合は扱う。

掲載対象は一般利用できる日帰り温浴施設とする。スーパー銭湯、日帰り温泉、浴槽利用を主目的にできるサウナ併設施設を中心に掲載する。

## 画面構成

### 地図

トップページは地図を主役にする。施設の位置を直感的に把握し、市区町村フィルター、現在地ボタン、ピン選択時の施設カードだけで候補を絞れるようにする。

地図画面で表示するもの:

- 温浴施設のピン
- 市区町村フィルター
- 現在地ボタン
- 選択した施設のカード
- リスト画面への導線

モバイルでは、地図上部に市区町村フィルターを置き、選択施設は下部パネルで表示する。Desktopでは地図の広さを活かし、選択施設カードを左側に表示する。

現在地はブラウザで許可された場合だけ利用する。距離表示や近い順の並び替えに使い、サイト側には保存しない。許可されない場合も、市区町村フィルターで探せる状態を保つ。

### リスト

`/list` は、地図なしで施設を比較できる一覧ページにする。地図が苦手な人や、Desktopで施設情報を見比べたい人のための補助導線。

リスト画面で表示するもの:

- 施設名
- 市区町村、エリア
- 住所
- 特徴タグ
- 現在地からの距離
- Google Mapsリンク
- 公式サイトリンク

現在地を許可した場合は近い順に並び替える。未取得時は、市区町村の優先順と施設名順で並べる。

### 利用規約

`/terms` に、掲載情報、現在地、アクセス解析、連絡先をまとめる。掲載情報の正確性は保証せず、営業時間、料金、休館日などは訪問前に公式サイトで確認してもらう。

## データ

施設情報はAstro Content Collectionで読み込む。`src/content/spas/` に1施設1JSONを置き、差分確認とPull Requestレビューをしやすくする。

現在の施設データ項目:

- `id`: 安定した英数字slug
- `name`: 施設名
- `city`: 市区町村
- `area`: 任意のエリア名
- `address`: 住所
- `officialUrl`: 公式サイトURL
- `googleMapsUrl`: Google Maps URL
- `lat`, `lng`: 緯度経度
- `tags`: 特徴タグ
- `notes`: 公開してよい補足
- `lastCheckedAt`: 確認日

内部メモ用の非公開項目は持たない。料金、営業時間、定休日、写真は変わりやすいため、初期設計ではデータ構造に含めない。必要になった段階で、確認頻度と表示責任を決めてから追加する。

市区町村の標準並び順は、松戸市、柏市、流山市、我孫子市、野田市、鎌ケ谷市を優先する。未定義の周辺市区町村は名前順で扱う。

## 技術方針

静的サイトとして軽く作る。地図部分だけ必要なJavaScriptを使い、サーバー、DB、ログイン、投稿機能は持たない。

採用技術:

- Astro v6
- TypeScript
- Leaflet + OpenStreetMap
- Astro Content Collection + JSON
- Cloudflare Workers Static Assets
- Vitest
- Prettier / ESLint

コアロジックは小さくテストする。特に、距離計算、市区町村の並び順、施設一覧の並び替えはVitestで確認する。

地図タイルはOpenStreetMap標準タイルから始める。アクセスが増えた場合は、利用規約や負荷を見て管理されたタイルサービスへの移行を検討する。

## コントリビュート

施設情報の追加・修正は、GitHub IssuesまたはPull Requestで受け付ける。画面上にはGitHubリポジトリと作成者Xへのリンクを置く。

参加しやすくするため、以下を用意する:

- `CONTRIBUTING.md`
- 施設追加・修正用Issueテンプレート
- Pull Requestテンプレート
- MITライセンス
